# lacon

Linear algebra in your Console — yayo!

lacon runs in your terminal and provides a set of linear algebra commands.
All operations are performed using [NumPy](https://github.com/numpy/numpy). 

**Status:** sections [0]–[4] of the [Feature](#feature) roadmap are implemented and reachable from the CLI — basic matrix operations, property checks, rotations, factorizations and least squares, all backed by NumPy in [`src/lacon/core`](src/lacon/core). I/O beyond headerless CSV (section [5]) is still a roadmap.

### Usage

Requires Python >= 3.14, NumPy and PyArrow.

Every feature is reached as `<command> <feature>`, loading its matrices from CSV files:

```sh
# multiply two matrices
python -m lacon.main bmo matmul -f A.csv -g B.csv

# LU decomposition, written to CSV
python -m lacon.main factorization lu -f A.csv -out lu.csv

# ridge least squares (lambda via -k)
python -m lacon.main least-squares regularized -f A.csv -g b.csv -k 1.0
```

Use `python -m lacon.main <command> <feature> --help` to list a feature's flags.

### I/O
You can set stdin and stdout to use files, pipes, or both in powerful combinations.

### Loading Pipeline

The loader delegates parsing to [PyArrow](https://arrow.apache.org/)'s `pyarrow.csv.read_csv`, configured for headerless numeric CSV: autogenerated column names, internal multi-threaded parsing, and no quoting/escaping for speed. The resulting Arrow table columns are stacked with `np.column_stack` into the final 2D NumPy array. See [`run`](src/lacon/main.py).

### Argument Parser

The parser is split by command (`bmo`, `checks`, `rotation`, `factorization`, `least-squares`), and each command exposes one sub-command per feature — every feature is reached through a command, never by direct access. Shared flags come from parent parsers: `-f/--file` (primary matrix), `-g/--file2` (second operand), `-k/--scalar`, `-b` (start-to-end benchmark) and `-out/--out` (CSV output path). See [`get_argparse`](src/lacon/arg.py).

### Output

When `-out` is given, the operation's result is written back as headerless CSV via PyArrow (loader-compatible). Multi-part results (e.g. `P`/`L`/`U` from a factorization, or the transform and transformed points from a rotation) are written one file per component as `<stem>.<name><suffix>`. See [`write_csv`](src/lacon/writer.py).
